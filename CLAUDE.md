## Project Overview

TempliMail is an email marketing web application (TFG/DAW project). It has an Angular 19 frontend and a pure PHP (no frameworks) backend, running on XAMPP locally with a MySQL database.

## Development Commands

### Frontend (Angular 19)

```bash
cd frontend
npm install          # install dependencies (first time)
ng serve             # dev server at http://localhost:4200
ng build             # production build
ng test              # run unit tests (Karma/Jasmine)
```

### Backend (PHP)

The backend runs automatically via Apache/XAMPP. No build step needed.

- API base URL: `http://localhost/TempliMail/backend/api/index.php`
- SMTP config: `backend/src/Utils/smtp_config.php`
- DB config: `backend/src/Utils/DB.php` (hardcoded: `localhost`, db `templimail_db`, user `root`, no password)

### Database

Import `templimail_db.sql` into phpMyAdmin as database `templimail_db`. Default admin credentials: `admin / admin123`.

## Architecture

### Frontend (`frontend/src/app/`)

- **Standalone components** — Angular 19 with no NgModules; all components use `standalone: true`.
- **Single service** — `services/api.service.ts` is the only Angular service; all HTTP calls to the backend go through it. JWT token is stored in `localStorage` and sent as `Authorization: Bearer <token>` on every protected request.
- **Two layout zones** — public routes (`/login`, `/register`) render without a shell; all authenticated routes render inside `layout/layout.component.ts` which includes the sidebar.
- **Lazy-loaded pages** — every page under `pages/` is lazy-loaded via `loadComponent` in `app.routes.ts`.
- **Scheduled sending** — polling for scheduled campaigns is triggered from the frontend (no server cron). `ApiService.processScheduledCampaigns()` calls `GET /process-scheduled`.

### Backend (`backend/`)

- **Single entry point** — `backend/api/index.php` handles all routing via manual `if` blocks and regex matches. No framework router.
- **Namespace** — all PHP classes use `TempliMail\` mapped via PSR-4 to `backend/src/`.
- **Auth flow** — JWT is generated by `JwtService` (HS256, 8-hour TTL) and validated by `AuthMiddleware`. The middleware sets `$_SERVER['AUTH_USER_ID']` so controllers can read the authenticated user's ID. Token version in DB prevents reuse of old tokens.
- **Layer structure**:
  - `Controllers/` — parse request body, call Services, return JSON responses.
  - `Services/` — business logic; static methods.
  - `Models/` — raw PDO queries; static methods.
  - `Utils/DB.php` — singleton PDO connection.
  - `Utils/Mailer.php` — wraps PHPMailer with SMTP config from `smtp_config.php`.
  - `middleware/AuthMiddleware.php` — JWT validation; exits with 401 on failure.
- **File uploads** — `.docx` parsing uses PHPWord, `.pdf` parsing uses smalot/pdfparser. Uploaded files are written to the `uploads/` directory at the project root.

### API Endpoints

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| POST | `/login` | Public | Returns JWT |
| POST | `/register` | Public | Creates user |
| GET | `/contacts` | JWT | List contacts |
| POST | `/contacts` | JWT | Create contact |
| PUT | `/contacts/:id` | JWT | Update contact |
| DELETE | `/contacts/:id` | JWT | Delete contact |
| GET | `/templates` | JWT | List templates |
| POST | `/templates` | JWT | Create template |
| PUT | `/templates/:id` | JWT | Update template |
| DELETE | `/templates/:id` | JWT | Delete template |
| POST | `/upload-template-file` | JWT | Parse .docx/.pdf to HTML |
| POST | `/send-mail` | JWT | Send single email |
| POST | `/send-massive` | JWT | Create campaign (immediate or scheduled) |
| GET | `/process-scheduled` | JWT | Process due scheduled campaigns |
| GET | `/history` | JWT | Email send history |

### Key Constraints

- The `.htaccess` in `backend/api/` rewrites the `Authorization` header because Apache/XAMPP doesn't pass it to PHP by default.
- CORS is set to `*` in `index.php` — only appropriate for local development.
- The JWT secret is hardcoded in `backend/api/index.php` (matches `backend/.env`). Both must stay in sync if changed.
- All models scope queries to `user_id` from `$_SERVER['AUTH_USER_ID']` — data is per-user.
